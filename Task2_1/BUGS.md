# Баги, найденные при тестировании API (Задание 2.1)

## 1. Метод создания объявления позволяет устанавливать отрицательную цену

- **ID:** BUG-API-001
- **Приоритет:** High
- **Серьезность:** Critical
- **Компонент:** `POST /api/1/item`
- **Версия API:** v1

### Описание:
Система некорректно обрабатывает значение поля `price` при создании объявления. Отправка запроса с отрицательным значением цены успешно проходит валидацию и создает объявление, возвращая статус-код `200 OK`. 

### Шаги для воспроизведения (Steps to Reproduce):
1.  Сформировать валидный JSON для создания объявления.
2.  В поле `price` установить любое отрицательное значение (например, `-100`).
3.  Отправить POST-запрос на эндпоинт `/api/1/item` с этим JSON-телом.

**Пример тела запроса:**
```json
{
  "sellerId": 123456,
  "name": "Товар с отрицательной ценой",
  "price": -100,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 1
  }
}
```
### Фактический результат:
- Запрос успешно обрабатывается.
- Сервер возвращает статус-код 200 OK.
- Тело ответа содержит сообщение об успешном сохранении:
```json
{
  "status": "Сохранили объявление - <UUID>"
}
```
### Ожидаемый результат:
- Запрос не должен проходить валидацию.
- Сервер должен вернуть статус-код 400 Bad Request.
- Тело ответа должно содержать информативное сообщение об ошибке, например:
```json
{
    "result": {
        "message": "Значение поля price не может быть отрицательным",
        "messages": {}
    },
    "status": "400"
}
```
